# EXP-011: Agent-Navigation A/B for the Intrinsic-ToC

**Date:** 2026-07-06
**Session:** 53 (Sprint 17 course correction, fork (c))
**Author:** Alberto Diaz Durana
**Status:** DESIGN (pre-run). Blocks the TOON redesign decision (fix-and-retry vs abandon).

Structure follows the revised 7-element standard in

## 1. Justification / Why

EXP-010 (Fable-5 assessment, Opus-adjudicated) surfaced finding **F8**: *nothing anywhere in
Epoch 5 tests whether an agent actually navigates a repository better or cheaper with the
Intrinsic-ToC than without it, in any format.* Sprint 17 spent 1–1.5 sessions optimizing the
ToC's **token cost** (the DEC-010 C3 gate) — a second-order proxy that the implemented TOON
schema currently **pessimizes** (F1: TOON is +1.74% to +7.58% tokens, where the gate needs
−10%). DEC-010's own counter-claim #1 concedes the token saving is ~0.7% of a 200K context; a
single unnecessary file-read by a disoriented agent costs more than the whole migration saves.

The gate it unlocks: the **Session-53 TOON redesign fork**. Before committing another sprint to
either fixing the TOON schema (fork a) or abandoning TOON (fork b), this experiment produces the
missing datum — does the ToC help navigation at all, and does the format matter to the consumer?
It also validates Intrinsic-ToC vision Layers 1–2 and de-risks Sprint 18 (Leiden clustering built
on the same topology).

## 2. Hypothesis

- **H1 (navigation value).** *If* an agent has an Intrinsic-ToC in context, *then* it answers
  fixed navigation questions with fewer tool calls and fewer tokens-to-answer at equal-or-better
  success than with no ToC, *because* the ToC pre-indexes where things live (files, sections,
  hub/hotspot/orphan structure).
  **Falsifier:** ToC arms (A1/A2) show no tool-call or success improvement over the no-ToC
  control (A0).

- **H2 (format).** *If* markdown-ToC and TOON-ToC are compared, *then* they perform comparably on
  success and tool calls, but TOON costs more context tokens (F1), so at equal navigation
  performance markdown wins on total cost.
  **Falsifier:** TOON (A2) yields materially better success or fewer tool calls than markdown
  (A1), which would justify its token premium and reframe the C3 gate.

- **Expected failure mode to watch:** the TOON arm mis-answers **T3** (orphan count) because the
  TOON schema drops overflow totals (F4) — a concrete, pre-registered prediction that TOON's
  information loss harms the consumer.

## 3. Success Criteria (pre-registered)

Measured per arm over the 8-task set (`tasks.md`): mean **tool_calls**, **success rate**, and
approximate **tokens_to_answer**.

- **"ToC helps navigation" (H1 supported)** ⇔ A1 and/or A2 cut mean tool_calls by **≥25%** vs A0
  at **≥** A0's success rate.
- **"Format matters" (H2 falsified)** ⇔ A2 vs A1 differ by **≥25%** on tool_calls, or show any
  success-rate gap. Otherwise treated as a tie → markdown wins on token cost.

Thresholds are directional gates for a small-N (n=8) capability experiment, not statistical
claims. n=8 buys a directional signal, not significance; a follow-up on DSM Central (larger
corpus) is the escalation path if the signal is ambiguous.

## 4. Environment

- **Corpus:** GE `dsm-docs/` — 116 files, 1650 sections, 67 cross-references (snapshot 2026-07-06).
- **Arm inputs (frozen):** `arm-inputs/toc-markdown.md` (2,903 `cl100k_base` tokens),
  `arm-inputs/toc-toon.toon` (3,123 tokens). Generated by `dsm-validate` (entry point
  `cli:main`; note `python -m cli` is a no-op — no `__main__` guard), relative-path invocation.
- **Toolchain:** Python 3.12, networkx 3.6.1, tiktoken `cl100k_base`, GE `--knowledge-summary`
  emitter at commit on branch `sprint-17/toon-migration`.
- **Run harness:** each (arm × task) = one fresh isolated subagent (Agent tool), tools limited to
  Read/Grep/Glob/Bash, no memory of this repo. Arms are blind: the subagent is not told which arm
  it is in; the only difference is whether/how the ToC text is prepended.
- **Measurement limits (honest):** `tool_calls` is counted directly and is the primary cost
  metric. `tokens_to_answer` is approximate — per-subagent token metering is weak — and is
  reported as best-effort context, not a gate input.

## 5. Results (post-run)

Full per-run detail in `results.md` (workflows `wf_74a16d93-706` pilot + `wf_a383617e-e17` batch,
24/24 agents). Rollup over all 8 tasks:

| Arm | mean tool_calls | success rate | notes |
|-----|-----------------|--------------|-------|
| A0 (no ToC) | 3.75 | 4/8 (+1 partial) | slower AND wrong more often; confident wrong answers on T1/T2/T5 |
| A1 (markdown) | 0.625 | **8/8** | correct on every task |
| A2 (TOON) | 0.625 | 7/8 | identical to A1 except **T3** (orphans 15 vs 112, F4) |

- **H1 confirmed:** ToC arms ~6× fewer tool calls and higher accuracy; the gain concentrates on
  graph-derived questions (hub/hotspot/orphans/section-count) the agent can't cheaply reconstruct.
- **H2 result:** markdown **strictly dominates** the current TOON — identical on 7/8 tasks, TOON
  wrong on T3 (F4 orphan-total loss), plus TOON costs more tokens (F1). No task favored TOON.
- Predicted failure mode (TOON mis-answers T3 via F4) **occurred exactly as pre-registered.**

## 6. Decision (post-run)

Pre-registered pivot/persevere rule (expected X, observed Y, conclusion Z per criterion):

1. **A1/A2 ≈ A0** → the ToC does not aid navigation → **abandon TOON** and re-examine the whole
   Intrinsic-ToC value proposition (escalates beyond Sprint 17).
2. **A1/A2 < A0 (ToC helps), and A2 ≤ A1** → **markdown wins** (cheaper tokens, F1) →
   deprioritize/abandon TOON; keep the ToC in markdown.
3. **A1/A2 < A0, and A2 ≫ A1** → TOON's structure genuinely helps navigation → **fork to
   fix-and-retry (a)**, with the navigation benefit justifying the token cost and reframing the
   C3 gate.

**Observed → Outcome 2** (A1/A2 ≪ A0; A2 ≤ A1). ToC arms ~6× fewer tool calls and higher
accuracy → the ToC helps (rule 1 rejected). TOON never beat markdown and lost on T3 → A2 ≤ A1.

**Gate outcome: NO-GO on the TOON migration; the Intrinsic-ToC is VALIDATED and kept in markdown.**
Confirmed by the author (S53). Fork (a) fix-and-retry not pursued (ceiling is a tie with markdown
while still owing an unlikely token win). Formalized in DEC-010 Amendment 2 and BL-302 Resolution.
