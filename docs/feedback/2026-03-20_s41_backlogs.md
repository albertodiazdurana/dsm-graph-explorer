# Session 41 Backlog Proposals

## Proposal #47: Add session branch creation step to /dsm-go and /dsm-light-go skills

**Date:** 2026-03-20
**Session:** 41
**Type:** Backlog Proposal
**Priority:** High
**Related:** Entry 55, Three-Level Branching Strategy (DSM_0.2 core lines 765-843)

**Problem:** The `/dsm-go` skill has no step that creates a Level 2 session branch. The Three-Level Branching Strategy in DSM_0.2 core declares this should happen at every session start, but the skill was never updated. As a result, all sessions run on `master`/`main`, defeating the isolation model.

**Proposed Solution:** Add a new step to `/dsm-go` (between the current Git Awareness and Step 1, or after Step 3.6 and before git status) that:

1. Checks for open branches from previous sessions (Session-Start Branch Resumption Protocol)
2. If an open session branch exists: resume on it, inform the user
3. If no open branch: create `session-N/YYYY-MM-DD` from current HEAD on main
4. All subsequent steps (baseline, transcript, etc.) operate on the session branch

The same logic should be added to `/dsm-light-go`, with the difference that `/dsm-light-go` resumes the existing session branch rather than creating a new one.

Also update `/dsm-wrap-up` and `/dsm-light-wrap-up` to merge the session branch back to main (Level 2 → Level 1) as part of wrap-up.

**Evidence:** Session 41 started on `master` despite 40+ prior sessions. The branching strategy has been in DSM_0.2 core but never enforced by the skills.
